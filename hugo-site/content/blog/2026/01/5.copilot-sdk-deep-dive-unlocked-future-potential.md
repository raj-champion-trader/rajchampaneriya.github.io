## The SDK: Putting the Agent to Work

Now, this is where it gets interesting. For a long time, we've been building AI into apps using heavy infrastructure like Microsoft Foundry. That's fine if you want to mortgage your soul to the cloud. But what if you just want to integrate the agent that powers the CLI directly into your own applications?

Enter the **Copilot SDK**. It stealth-launched into technical preview, and it is exactly what you need if you are serious about leverage.

It's simple. The SDK—available in Node, Python, Go, and C#—allows you to communicate programmatically with the Copilot CLI via JSON-RPC. Your app acts as the client; the CLI runs in server mode. You get full session management, access to models across all providers (GPT, Claude, Gemini—take your pick), tool calling, and streaming results.

It's madness, really. We used to have to write code to get code. Now we write code to ask a machine to ask another machine to write the code.

## A Practical Application: The Metadata Grind

Let's look at a concrete example. Like many of you, I generate content. I have transcripts. I need titles, descriptions, and YouTube chapters. Doing this manually is tedious. Doing it with generic LLM prompts is hit-or-miss.

So, I built a **Podcast Metadata Generator** using the Copilot SDK.

I built a console app and a Blazor web application in under an hour. I'm not exaggerating—I barely touched the keyboard. The code connects to the local Copilot CLI via the SDK, loads a transcript, and executes a specific set of instructions—what the kids are calling "Claude Skills" these days, but we'll stick to "prompts" for now.

**Input:** I dump a long transcript into the app.  
**Process:** The app sends it to Copilot.  
**Output:** Titles, descriptions, and time-stamped chapters flow back in real-time.

I used the `github.copilot.sdk` NuGet package. I started a session with a specific model, pinged it to ensure a connection, and streamed the results. I didn't have to manage API keys. I didn't have to worry about token limits draining my credit card directly. I just used my existing GitHub subscription.

It's beautiful in a terrifying way. I can now generate a week's worth of content metadata while I'm busy doing something else, like pretending to care about meeting agendas.
