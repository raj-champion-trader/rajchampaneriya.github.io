# Book Summaries & Reviews

Practical insights from books, mapped to real work experience.

---

## Purpose

Book reviews that are **useful**, not just summaries:
- Key insights that changed how you work
- Practical applications from actual projects
- Honest assessment of what's useful vs. theoretical
- Recommendations for specific situations/roles

This is NOT:
- Chapter-by-chapter summaries
- Generic book reports
- Affiliate marketing disguised as reviews

---

## Book Review Template

See [`/backlog/07-automation/templates/blog-post-template.md`](../07-automation/templates/blog-post-template.md) as base structure.

### Key Elements to Include

#### 1. Book Details
- Title, Author, Publication Year
- Who should read this
- Your context when reading (role, situation)

#### 2. Core Ideas
- 3-5 main concepts
- Brief explanation of each
- Why each matters

#### 3. What Resonated
- Ideas that matched your experience
- Concepts you've applied successfully
- "This explains what I've been seeing" moments

#### 4. What You Questioned
- Ideas that seem theoretical
- Concepts that don't scale/work in practice
- Context where book's advice breaks down

#### 5. Practical Applications
- How you've used these concepts at work
- Specific examples (anonymized)
- Results or outcomes

#### 6. Who Should Read This
- Specific roles (e.g., "architects moving to principal level")
- Specific situations (e.g., "teams transitioning to microservices")
- Prerequisites (e.g., "needs 5+ years of system design experience")

#### 7. Key Takeaways
- 3-5 actionable insights
- Memorable quotes
- Your one-sentence summary

## Review Frequency

**No fixed schedule** - Review books as you read them.

Ideal cadence:
- 1 technical book every 2-3 months
- Review within 1 week of finishing
- Re-read and update notes quarterly

---

## Output Format

- **Blog Post**: Full review published to blog
- **LinkedIn**: Key insights (3-5 bullet points)
- **Reference**: Personal notes in this folder

---

## Success Criteria

A good book review:
✅ Helps someone decide if they should read the book  
✅ Provides value even if they don't read the book  
✅ Includes your unique perspective and experience  
✅ Is honest about limitations and context  
✅ Actionable - reader can apply insights immediately  

---

*Books are intellectual capital - document the return on investment.*
