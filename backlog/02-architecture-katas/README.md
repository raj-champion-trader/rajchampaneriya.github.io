# Architecture Katas

System design exercises that demonstrate architectural thinking.

## What Are Architecture Katas?

Structured problem-solving exercises that show:
- How you analyze requirements
- How you identify trade-offs
- How you make architectural decisions
- How you document your reasoning

## Value Proposition

This content type is **gold** for:
- Interviews for senior architect roles
- Demonstrating senior-level thinking
- Standing out from typical blogs

## Kata Structure

Each kata should include:

1. **problem.md** - The scenario and requirements
2. **assumptions.md** - What you're assuming and why
3. **tradeoffs.md** - Key decisions and alternatives considered
4. **final-architecture.md** - Your solution with diagrams
5. **lessons.md** - What you learned from the exercise

## Example Topics

- Design a payment processing system
- Build a scalable notification service
- Create a multi-tenant data architecture
- Design for zero-downtime deployments
- Handle eventual consistency in distributed systems

## Naming Convention

`kata-NNN-topic-name/`

Example: `kata-001-payment-system/`

---

**Frequency**: 1 per month (Week 2 of rotation)
**Time Investment**: 2-3 hours per kata
**Output**: Blog post + diagrams + GitHub folder
